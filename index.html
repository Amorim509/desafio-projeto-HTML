Comunicação client x server

O que são clients

(voltar)

Client é tudo aquilo presente do lado do usuário, então qualquer dispositivo com acesso ao seu programa é um client, seja ele um notebook, um desktop ou até mesmo um smartphone. Osclients através da internet buscam conteúdos presentes nos servidores, e estes retornam os arquivos daquilo que foi buscado(HTML, CSS, Imagens...).

Os navegadores são programas criados por empresas, que são utilizados para abrir e executar esses arquivos, eles fazem parte daquilo que chamamos de client.

Aplicações web são soluções criadas que possuem a internet como meio de comunicação entre client x server não sendo necessário sua instalação, ou seja, são os sites que temos acesso pelo nosso navegador.

O que são servers

(voltar)

Servidores são computadores que servem pra armazenar as informações de ua aplicação. Eles possuem um sistema operacional diferente do que estamos acostumados, como o windows server, Ubuntu, CentOs... Existem servidores paravários tipos de aplicação, como firewalls, bancos de dados e aplicações web.

Os servidores são responsáveis pela hospedagem dos sites, a partir do momento em que um site está hospedado em um servidor oclient pode fazer uma requisição de acesso, e então o servidor retorna os arquivos requeridos, que por fim são interpretados pelos navegadores

Linguagens de programação

(voltar)

É uma linguagem escrita e formal que especifica um conjunto de regras e instruções usadas para gerar um programa (software). Existem várias linguagens, cada uma com sua peculiaridade, e dentre elas costuma-se separar as linguagens usadas para o client-side (front-end), e as usadas para o server-side (back-end).

Dentro das linguagens de client-side se destaca o Javascript, existem outras que também funciona, mas o javascript é o que mais se destaca, principalmente em aplicações web. Uma curiosidade é que javascript roda tanto no client-side quanto no server-side. Além disso o client-side também vai utilizar de elementos HTML e CSS (que não são bem linguagens de programação)

Dentro das linguagens de programação do server-side temos uma variedade muito maior, como PHP, C#, Java... Essas linguagens não vão rodar do lado do client (ou seja no equipamento do usuário), e sim nos servidores, cada uma delas tem sua peculiaridade, não que uma seja melhor que a outra, mas para algumas explicações algumas se tornam mais eficientes que outras.

Introdução do HTML na prática

Estrutura básica HTML

(voltar)

A estrutura básica HTML é iniciada sempre pela tag html, que delimita o início e o fim do nosso arquivo HTML. Logo em seguida temos a tag head que vão ser informações que vão carregar antes de exibir para o usuário como o título por exemplo. E por fim a tag body, que indica o que realmente vai ser mostrado para o usuário.

Tags

(voltar)

É forma utilizado pelo HMTL para dizer o que deve ser feito. Um exemplo é a tag i, que indica que tudo entre o fechamento e abertura dessa tag deve ficar em itálico. As tags são escritas pelo sinal <>.As tags também possuem seus atributos que são propriedades que tags podem ter, alguns atributos são mais globais, como o style, que coloca comandos CSS direto no HTML, já alguns atributos são específicos de suas tags

Textos

(voltar)

Existem tags específicas para inserr textos na nossa página. As tags h1, h2, h3... servem basicamente como cabeçalhos, quanto menos o número após o h, menor será a fonte. Atag p é usada para escrever textos maiores dentro do site. e também a tag blockquote que insere citações ao site. Esses foram alguns exemplos de implementação de textos.

Listas ordenadas e não ordenadas

(voltar)

São usados para listar elemntos presentes no site, como um índice. A grande dferença entre as duas é a presença de um contador numérico nas listas ordenadas. São representadas pela tag ul para listas não ordenadas, e ol para as listas ordenadas.
